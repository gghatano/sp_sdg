# DS-5 実行価値の批判的評価 — マルチモーダル/チャネル横断拡張は「今やる価値があるか」

- 作成日: 2026-07-20 / レーン: Research / Design / 対応 issue: #21 DS-5(#7 候補8「被験者内拡張」と接続、#23 レッドチームの教訓を反映)
- 位置づけ: **実行価値の誠実な判断が最重要成果物**。実装・run はしない。裏取りは §3 に URL 併記。

---

## 0. 結論(推奨: 後回し=現状では実質不要)

**推奨: 後回し。現状の測定基盤では実質的に「不要」に近い。**

理由を一文で: DS-5 が問う「チャネル横断拡張が**被験者数削減に効くか**」を測るには、まず **N\*(none) が被験者格子内に落ちて削減率が推定できる基盤**が要る。ところが DS-5 の想定対象である多モーダルデータは、**PAMAP2 が左側打ち切りで削減推定不能(F-13/DS-2)**、**WESAD が near-chance でモデル未学習(F-14/DS-3)**。両者とも「削減が測れる余地」自体を欠くため、その上にマルチモーダル拡張を載せても得られるのは「また推定不能/帰無」で、**新拡張手法と多チャネル入力の実装コストに見合う新規知見が出る見込みが低い**。しかも #21 で【低】優先。

したがって:
- **今やる → 非推奨**。実装コストを払っても defensible な結論の上限が「帰無/推定不能の5例目」に留まる。
- **条件付きで将来やる価値はある**: ただし「多モーダルデータで none baseline の N\* が推定可能(モデルが実際に学習し、N\* が格子内)」という**前提を先に満たした場合に限る**。その前提は現状どのデータでも未成立で、成立させること自体が別の大仕事(§2)。
- **#21 を締めるには、DS-5 を実行せずに** §4 の代替(4データ統合まとめ + 前処理 ablation の別issue化)で閉じるのが費用対効果で優る。

以下、論点ごとに根拠。

---

## 1. DS-5 が意味を持つ前提が成立するデータはあるか — **無い**

DS-5 の主旨は「**モダリティ横断**(センサ間 mixup 等)の拡張が被験者数削減に効くか」。これが測定として成立する条件は2つ:(A) 複数モダリティ/センサ位置を持つ多チャネルデータであること、(B) その上で **none baseline の削減率が推定可能**(N\* が格子内・モデルがタスクを学習)であること。既存4データを両条件で照合する。

| データ | モダリティ多重性 (A) | 削減推定可能性 (B) | DS-5 基盤としての可否 |
|---|---|---|---|
| **PAMAP2**(3 IMU=hand/chest/ankle、accel/gyro/mag) | ◯ 位置×種別で多重。DS-5 の「センサ間」に最も合致 | **✗ 左側打ち切り**。none macro-F1 が最小格子 N=2 で既に target 超過、削減率**推定不能**(F-13, reduction_pamap2) | **不適**。拡張差を測る土台が無い |
| **WESAD**(胸生理 ECG/EDA/EMG/RESP/TEMP + ACC、手首 E4) | ◎ 真の異種モダリティ。DS-5 の主旨に最適 | **✗ near-chance**。HAR 枠組み(窓ごと z 正規化+cnn1d_har)が非転移、label_shuffle≈none でモデル未学習、全手法 N\* 左側打ち切り(F-14) | **不適**。モデルがそもそも学習していない |
| **UCI HAR**(単一 IMU、accel+gyro 6ch) | △ 単一モダリティに近い(慣性のみ、加速度+ジャイロ) | ◯ N\* 推定可(none≈8.85名、F-8) | 削減は測れるが**モダリティ横断の主旨と不整合**(§下記) |
| **WISDM**(単一 accel 3ch) | ✗ 単一モダリティ・単一種別 | ◯ N\* 推定可(none≈14名、F-12) | センサ間 mixup の対象たり得ない |

**帰結**: DS-5 の「モダリティ横断」の主旨に合う2データ(PAMAP2/WESAD)は**いずれも削減が測れない**。削減が測れる2データ(UCI HAR/WISDM)は**モダリティ横断の主旨に合わない**。

- UCI HAR の 6ch(accel×3 + gyro×3)を「チャネル横断」の対象と読み替えることは技術的には可能だが、加速度とジャイロは**物理量が異なり単位・スケール・意味が違う**。両者を mixup/shuffle で混ぜる操作は「モダリティ横断拡張が個体間分散を代替するか」という問い以前に、**信号として意味の壊れた合成**になりやすく、defensible な主張に繋がりにくい(F-3 の「MiniRocket で振幅・ノイズに敏感」と同型のリスク)。WISDM の accel 3ch(x/y/z)は同一種別の軸なので「センサ間 mixup」ですらない。
- すなわち **DS-5 の前提を「今」満たすデータは、既存4データには存在しない**。

---

## 2. defensible な主張の上限 — 「帰無/推定不能の5例目」に留まる

- これまで **削減自体が全4データで帰無ないし推定不能**(F-8/F-10=対照越え不能、F-12=WISDM 帰無、F-13=PAMAP2 打ち切り、F-14=WESAD 未学習)。この状況で「マルチモーダル拡張が削減に**効く**」を新たに示せる現実的見込みは**低い**。効くとしても、単一チャネル拡張(mixup/dtw/smote)ですら対照(label_shuffle/oversample)を明確に超えられなかった(F-10)以上、チャネル横断版だけが対照越えを達成する事前確率は小さい。
- 「効果を示せなくても帰無の追加証拠になるか」: 原理的にはなるが、**限界効用が小さい**。既に4データ・2信号種で帰無/推定不能が揃っており、5例目(しかも測定基盤を欠く)を足しても外的妥当性の増分は乏しい。むしろ「基盤が無いのに手法を増やした」という**設計の緩みと読まれるリスク**(#23 の過大主張回避・post-hoc 回避の規律に照らして減点方向)。
- 最悪ケース: PAMAP2/WESAD 上で走らせれば **DS-2/DS-3 と同じく「左側打ち切り/near-chance で削減推定不能」** を再生産するだけ。手法(none vs 各拡張)の比較以前に土台が崩れているので、**マルチモーダル拡張の寄与を分離できない**。これは「DS-5 をやったが何も言えなかった」という最も価値の低い帰結。

## 3. 技術的設計(仮に実行する場合の骨子)

将来 §0 の前提が満たされた場合に備え、設計だけ記録する(**今の推奨は実行しない**)。既存フレームへの載せ方は DS-1〜3 と完全同型。

### 3.1 チャネル横断拡張の具体(新 augmenter 案)
既存の `augment(X, y, rng, **params) -> (X_out, y_out)`(`X`=`(n, channels, length)`)に、**チャネル群のメタ情報を渡す**版を追加する必要がある(既存拡張はチャネル非依存で動くが、DS-5 拡張は「どのチャネルがどのモダリティ/センサ位置か」を要求する)。config の `dataset_params.channel_groups`(例 PAMAP2: `{hand:[0..2], chest:[3..5], ankle:[6..8]}`)を manifest 経由で渡す。

1. **sensor-mixup(センサ間 mixup / センサ再結合)**: 同一クラスの2サンプル `i, j` から、**センサ位置ブロック単位**で凸結合または入れ替え(例: chest ブロックは `λX_i + (1−λ)X_j`、他は `X_i`)。ラベルは同一クラスで保存(既存 mixup と同じソフトラベル非生成方針)。「個体 i の chest と個体 j の chest を混ぜて個体間分散を合成」= H-CORE の被験者内→被験者間橋渡しの直球。
2. **channel/sensor recombination(センサ再結合)**: 同一クラスの複数サンプルから **センサ位置ブロックを別サンプル由来で寄せ集め**て1サンプルを合成(i の hand + j の chest + k の ankle)。物理的整合は崩れるが「センサ横断の新規組合せで多様性を稼ぐ」意図。**軸単位のシャッフルは禁止**(x/y/z や accel/gyro を跨ぐと信号が壊れる=§1 の懸念)。ブロック単位に限定するのが defensible。
3. **modality dropout**: 学習時に**モダリティ/センサ位置ブロックを確率的にゼロ化**した複製を合成(欠損センサへの頑健化。文献では robustness 目的が主で削減目的ではない点に注意=§3.3)。

いずれも **同一クラス内・ブロック単位・ラベル保存**を厳守(#23: 意味を壊さない、post-hoc 回避)。

### 3.2 既存フレームへの載せ方(DS-1〜3 と同型)
- `augmentations/methods.py` の `REGISTRY` に `sensor_mixup` / `sensor_recombine` / `modality_dropout` を追加。ただし**チャネル群情報の受け渡し**のため `apply_augmentation` に `channel_groups` を通す小改修が要る(既存署名の後方互換は保つ)。
- 対象データは **前提充足が確認できた多モーダルデータのみ**。既存 `subject_datasets` dispatch(UCI_HAR/WISDM/PAMAP2/WESAD)に新規追加は不要で、config の augmentation 列に新手法を足すだけ。
- 被験者数削減グリッド・reduction 集計・横断図・negative control(label_shuffle)・両指標(macro-F1/accuracy)併記は **DS-2/DS-3 と同一**。
- **事前登録項目**(結果を見る前に固定): target 選択ルール(full-pool none −0.05 切り捨て、既存と同一)、`channel_groups` の定義、対象拡張手法の集合、格子・反復・split seed、主指標。加えて **DS-5 固有の事前条件**として「full-pool none baseline が N\* 推定可能マージン(≥0.05)を満たさなければグリッドを回さず停止」を登録(§0 前提の明文化)。
- **停止条件(spec §11)**: (a) none baseline が左側打ち切り/near-chance なら **削減推定不能を明記して停止**(PAMAP2/WESAD の前例どおり)。(b) チャネル横断合成が信号を壊し baseline より劣化するだけなら記録して停止。(c) 同一失敗2回で停止。

### 3.3 #23 の教訓の反映
- **過大主張回避**: マルチモーダル拡張は文献上**頑健化・センサ融合**が主目的で、「被験者数削減」を主張した先行研究は本調査では確認できず(§下記の裏取り)。効果が出ても「削減」への外挿は慎重に。
- **test 禁止**: チャネル群定義・拡張パラメータ・手法選択に test を使わない。
- **post-hoc 回避 / 両指標**: 不均衡データは macro-F1 主・accuracy 併記、target は数値確定前に手続き登録。
- **対照**: label_shuffle(悲観対照)+ oversample(純量水増し基準)を DS-5 でも並置。チャネル横断拡張が oversample を超えるかが最低ライン。

### 裏取り(一次情報)
- チャネル横断/マルチモーダル mixup・modality dropout はウェアラブル HAR で**確立した拡張系統**だが、**目的は頑健化・センサ融合・欠損モダリティ耐性**であり、「必要**被験者数**の削減」を検証した研究は本調査では**確認できず(不明)**。
  - 多モーダル HAR サーベイ(mixup を data-agnostic な多モーダル拡張として位置づけ): *A Survey on Multimodal Wearable Sensor-based Human Action Recognition*, arXiv:2404.15349 <https://arxiv.org/html/2404.15349v1>。
  - modality dropout の頑健化目的の例: *Federated Learning for Time-Series Healthcare Sensing with Incomplete Modalities*, arXiv:2405.11828 <https://arxiv.org/pdf/2405.11828>;WESAD 上のクロスモダリティ検討 arXiv:2502.18733 <https://arxiv.org/html/2502.18733v1>。
  - PAMAP2 = 3 IMU(hand/chest/ankle)×(accel/gyro/mag)の多モーダル構成、WESAD = 胸 ECG/EDA/EMG/RESP/TEMP + ACC + 手首 E4 の異種モダリティ、は原データ仕様どおり(DS-2 §1 / DS-3 §1 で裏取り済み)。

## 4. コスト vs 期待価値 と 代替案

### コスト(実行する場合)
- **新拡張手法3種の実装 + テスト**、`apply_augmentation` へのチャネル群受け渡し小改修(既存署名の後方互換維持ぶんの検証コスト)。
- **channel_groups メタの config/manifest 配管**(既存拡張はチャネル非依存なので新規の横串)。
- **run コスト**: DS-2/DS-3 と同規模(~150–315 runs/データ)。ただし前提未成立のデータで回すと**産物は「推定不能」**。
- 期待価値: §2 のとおり上限が「帰無/推定不能の5例目」。**コスト > 期待価値**。#21 で【低】優先である事実とも整合。

### 推奨: 後回し。#21 を締める代替(DS-5 を回さずに閉じる)
1. **既存4データの統合的まとめ(最有力・低コスト)**: UCI HAR / WISDM / PAMAP2 / WESAD の削減評価を**1つの統合節**に束ね、「運動信号3データ(母集団 5–36名)で削減は帰無 or 推定不能、生理信号1データは枠組み非転移」という**外的妥当性の全体像**を1図1表で提示。#21 の DS-1〜3 の到達点を締めくくる論文価値が高く、新規 run 不要(既存 results.json から集計)。DS-5 の「マルチモーダルも試すべきでは」への回答も、この統合節に「多モーダルデータは削減の測定基盤を欠くため対象外」と1段落で明記すれば足りる。
2. **前処理 ablation を別 issue 化**: DS-5 より価値が高い未解決課題は、**WESAD の near-chance を生む前処理(窓ごと z 正規化が EDA/体温の絶対レベルを消す仮説、F-14 / J-WESAD-NORM)**。ここを ablation する別 issue(生理信号向け正規化=チャネルごと大域 z / min-max、モデル)を立てれば、「WESAD で削減が測れる基盤」を作る第一歩になり、**将来 DS-5 の前提(§0 条件)を満たす道**にもなる。これは DS-5 そのものより優先度が高い。
3. **(将来 DS-5 をやる場合の発火条件を明記)**: 「多モーダルデータで none baseline の N\* が格子内かつモデルが学習(label_shuffle≪none)を確認できたら、§3 の設計で DS-5 を起動」を #7/#21 のバックログに条件付きで残す。前提未成立のまま着手しないことを明文化。
